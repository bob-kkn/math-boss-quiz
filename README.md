# 수학 보스전 웹앱

React 기반 수학 퀴즈 게임 MVP입니다. 현재 기준은 본편에서 “8문항 = 1스테이지”, “10스테이지 = 1단계”이며, 각 본편 단계의 마지막 스테이지는 보스전입니다.

## 목표

- 1차 목표: 브라우저에서 플레이 가능한 MVP
- 핵심 구조: 본편 17단계, 단계별 10스테이지, 스테이지별 8문항
- 보너스 구조: 마지막 18단계, 1스테이지, 10문항
- 전체 문항: 본편 1,360문항 + 보너스 10문항 = 1,370문항 생성형 구조
- 보스전: 본편 각 단계의 10번째 스테이지 전체를 틀리기 쉬운 응용/심화 문제로 구성
- 중복 방지: 문제 앞 라벨 없이 본문 숫자와 조건을 달리해 같은 문장이 반복되지 않게 구성

## 단계 구성

아기, 어린이집, 유치원, 초1, 초2, 초3, 초4, 초5, 초6, 중1, 중2, 중3, 고1, 고2, 고3, 대학교, 대학원, 보너스

보너스 단계는 마지막 18단계이며, 거의 난제 성격의 10문제짜리 단일 스테이지로 구성합니다.

## 현재 MVP 범위

- 본편 17단계와 단계별 10스테이지 생성
- 본편 각 스테이지 8문항 생성
- 보너스 1스테이지 10문항 생성
- 4지선다, 숫자 입력, 참거짓 답안 유형
- 정답 제출과 점수 증가
- 콤보와 최고 콤보 기록
- 스테이지별 1~3별 보상 기록
- 보스전 전용 보스 HP, 플레이어 HP, 재도전 흐름
- 다음 문제, 다음 스테이지, 다음 단계 진행
- 각 단계 마지막 스테이지 보스전 표시
- 로컬 스토리지 기반 이어하기
- 모바일 폭에서 사이드바 진행 정보 확인

## 보스전 규칙

- 본편 각 단계의 10번째 스테이지는 보스전입니다.
- 보스 HP는 6, 플레이어 HP는 3으로 시작합니다.
- 정답이면 보스 HP가 1 줄고 콤보가 증가합니다.
- 오답이면 플레이어 HP가 1 줄고 콤보가 0으로 초기화됩니다.
- 플레이어 HP가 0이 되거나 8문항 종료 시 보스 HP가 남아 있으면 보스전 재도전 상태가 됩니다.
- 재도전 시 해당 보스 스테이지의 1번 문제로 돌아가며, 보스전에서 얻은 점수는 시작 시점으로 되돌립니다.

## 실행

```bash
npm install
npm run dev
```

## 검증

```bash
npm test
npm run build
npm run verify:prod
```

## 배포

Vercel 배포 기준:

- Framework Preset: Vite
- Build Command: `npm run build`
- Output Directory: `dist`
- Install Command: `npm install`

배포 전 확인:

```bash
npm test
npm run verify:prod
```

`npm run verify:prod`는 TypeScript 검사와 Vite 프로덕션 빌드를 실행하고, 개발 QA 버튼 문구가 번들에 남아 있지 않은지 확인합니다.

## QA 체크리스트

배포 전 [production-qa.md](docs/production-qa.md)를 기준으로 단계 진행, 스테이지 진행, 보스전, 저장/이어하기, 모바일 화면을 확인합니다.
