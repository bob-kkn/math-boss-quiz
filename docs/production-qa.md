# Production QA Checklist

배포 전 아래 항목을 확인합니다.

## 자동 검증

- `npm test`가 통과한다.
- `npm run verify:prod`가 통과한다.
- `dist` 번들에 `개발 QA`, `최종 보스 바로가기`, `보너스 바로가기`, `QA 초기화` 문구가 없다.

## 핵심 플레이 흐름

- Stage 1에서 선택지를 고르기 전 `제출` 버튼이 비활성화된다.
- 정답 제출 시 점수가 증가하고 해설이 나온다.
- 오답 제출 시 점수가 유지되고 해설이 나온다.
- 마지막 문제 후 다음 스테이지로 이동할 수 있다.
- Stage 13 마지막 문제는 `최종 보스` 배지가 붙는다.
- 최종 보스 정답 후 보너스 레벨 진입 버튼이 보인다.
- 보너스 5문제 완료 후 칭호가 보인다.

## 저장과 이어하기

- 문제 풀이 도중 새로고침해도 현재 스테이지와 문제 위치가 유지된다.
- `저장 초기화`를 누르면 Stage 1 첫 문제로 돌아간다.
- 손상된 저장 데이터가 있더라도 앱이 Stage 1에서 정상 시작한다.

## 모바일 화면

- 375px 폭에서 문제, 선택지, 제출 버튼, 다음 문제 버튼이 잘리지 않는다.
- 13단계 스테이지 목록은 사이드바 안에서 내부 스크롤로 확인된다.
- 숫자 입력 문제에서 모바일 숫자 키보드가 열린다.
- 긴 문제 문장이 카드 밖으로 넘치지 않는다.

## 배포 후 확인

- 배포 URL에서 첫 화면이 바로 퀴즈 플레이 화면이다.
- 브라우저 콘솔에 앱 실행 오류가 없다.
- 새로고침 후 저장된 진행이 유지된다.
- 개발 QA 버튼이 화면에 보이지 않는다.
