# Production QA Checklist

배포 전 아래 항목을 확인합니다.

## 자동 검증

- `npm test`가 통과한다.
- `npm run test:e2e`가 통과한다.
- `npm run export:content`가 통과한다.
- `npm run verify:prod`가 통과한다.
- `dist` 번들에 `개발 QA`, `최종 보스 바로가기`, `보너스 바로가기`, `QA 초기화` 문구가 없다.
- `dist` 번들에 필수 사용자 문구와 참조된 정적 자산이 남아 있다.
- `index.html`에는 description, theme color, OG 이미지, favicon, manifest가 포함된다.

## 게임 구조

- 전체 단계는 18개다.
- 단계 이름은 아기, 어린이집, 유치원, 초1, 초2, 초3, 초4, 초5, 초6, 중1, 중2, 중3, 고1, 고2, 고3, 대학교, 대학원, 보너스 순서다.
- 본편 17단계는 각각 10개 스테이지를 가진다.
- 본편 각 스테이지는 8문항을 가진다.
- 보너스 단계는 18번째 단계이며 1개 스테이지, 10문항만 가진다.
- 전체 생성 문항 수는 본편 1,360문항 + 보너스 10문항 = 1,370문항이다.
- 생성된 문제 문장은 전체에서 중복되지 않는다.
- 모든 문항에는 핵심 개념, 스킬 태그, 난이도 구간이 있다.
- 해설에는 `핵심:` 힌트가 포함된다.
- 저연령 단계 일반 스테이지에는 미분, 적분, 로그, 제곱근, 방정식 같은 고급 표기가 없다.
- 보스전은 응용/심화 문제이므로 직접입력 답안을 허용한다.
- 본편 각 단계의 10번째 스테이지는 보스전으로 표시된다.
- 보너스 단계는 거의 난제 성격의 문제로 구성된다.
- `reports/content-audit.csv`에서 단계, 스테이지, 문제, 정답, 선택지, 해설을 검수할 수 있다.
- 콘텐츠 검수 요약에서 전체 문항 수는 1,370개, 이슈 수와 중복 문항 수는 0이다.

## 핵심 플레이 흐름

- 첫 화면은 바로 퀴즈 플레이 화면이다.
- 선택지 또는 답안을 입력하기 전에는 `제출` 버튼이 비활성화된다.
- 정답 제출 시 점수가 증가하고 해설이 나온다.
- 오답 제출 시 점수는 유지되고 해설이 나온다.
- 정답 제출 시 콤보가 증가한다.
- 오답 제출 시 콤보가 0으로 초기화된다.
- 제출 전에는 다음 문제로 이동할 수 없다.
- 8번째 문제 제출 후 다음 스테이지로 이동할 수 있다.
- 10번째 스테이지 완료 후 다음 단계로 이동할 수 있다.
- 각 단계의 10번째 스테이지에는 보스전 배지와 보스전 피드백이 보인다.
- 문제 패널에는 핵심 개념과 난이도 구간 배지가 보인다.
- 보스전에서는 보스 HP와 플레이어 HP가 보인다.
- 보스전 정답 시 보스 HP가 1 줄어든다.
- 보스전 오답 시 플레이어 HP가 1 줄어든다.
- 플레이어 HP가 0이 되면 보스전 재도전 화면이 보인다.
- 보스전 재도전 시 해당 보스 스테이지의 1번 문제로 돌아간다.
- 스테이지 클리어 화면에는 별점, 정답 수, 최고 콤보가 보인다.
- 클리어한 스테이지 목록에는 획득한 별점이 보인다.
- 대학원 10번째 보스 스테이지 완료 후 보너스 단계로 이동할 수 있다.
- 보너스 10문항을 완료하면 전체 클리어 화면이 보인다.

## 저장과 이어하기

- 문제 풀이 중 새로고침해도 현재 단계, 스테이지, 문제 위치가 유지된다.
- 저장 데이터에는 콤보, 보스 HP, 플레이어 HP, 스테이지 별점 기록이 포함된다.
- 저장 상태 영역에 현재 위치와 완료 스테이지 수가 표시된다.
- `저장 초기화`는 한 번 더 확인해야 실제 초기화된다.
- `QA 초기화`를 누르면 아기 1스테이지 첫 문제로 돌아간다.
- 잘못된 저장 데이터가 남아 있어도 앱은 아기 1스테이지에서 정상 시작한다.

## 접근성

- 키보드 포커스 시 `퀴즈로 바로가기` 링크가 보인다.
- 현재 스테이지 진행률, 보스 HP, 플레이어 HP가 보조기술용 진행률로 노출된다.
- 버튼과 입력창의 키보드 포커스 위치가 눈에 띈다.
- 시스템에서 동작 줄이기를 켠 경우 선택지 움직임이 줄어든다.

## 모바일 화면

- 375px 폭에서 문제, 선택지, 제출 버튼, 다음 문제 버튼이 잘리지 않는다.
- Playwright 모바일 프로젝트에서 첫 문제 풀이와 보스전 재도전 흐름이 통과한다.
- 18단계 목록은 사이드바 안에서 내부 스크롤로 확인된다.
- 본편 단계의 10스테이지 목록은 사이드바 안에서 내부 스크롤로 확인된다.
- 보너스 단계에서는 1스테이지 목록만 보인다.
- 숫자 입력 문제에서 모바일 숫자 키보드가 열린다.
- 긴 문제 문장이 화면 밖으로 넘치지 않는다.

## 배포 후 확인

- 배포 URL에서 첫 화면이 바로 퀴즈 플레이 화면이다.
- 브라우저 콘솔에 실행 오류가 없다.
- 새로고침 후 저장된 진행이 유지된다.
- 개발 QA 버튼은 화면에 보이지 않는다.
- favicon과 앱 미리보기 메타데이터가 정상 로드된다.
